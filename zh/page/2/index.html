2019-Modeling Human Cancer-induced cachexia

2022年10月05日
文献笔记
1379 words 3 min read

今后准备每天都会在这里上传一篇文献笔记。每天写一篇吧，好歹给自己的博士生涯一个交代。并且要常常阅读。

背景

在肠癌患者中，尤其是胰腺导管癌（Pancreatic ductal adenocarcinoma, PDA）患者中，70%的人在诊断时就能够表现出恶液质。过去的认知认为只有到了癌症晚期才会出现恶病质的症状。因此作者想要建立更好的模型来模拟这种状况。

目前最常用的两种模型

  • Colon-26： C-26模型：移植瘤模型，需要在小鼠皮下种植C-26细胞系。
  • Lewis Lung carcinoma cell line

很明显，这两种模型的不足之处在于：

  • 不是自发肿瘤模型，由于需要皮下种植，因此不能研究肿瘤微环境对于恶液质发生的影响。
  • too aggressive, 发生恶病质的时间窗口太短，肿瘤进展太快了。

Summary

这篇文章比较了C-26模型，LLC模型以及PDA患者发生恶液质后骨骼肌的基因表达情况（这个后续自己要走一遍流程）。结果发现overlap很少。

作者还验证了经典的PDA模型，Kras; p53; Cre (KPC)，发现它在研究cachexia也有很大局限。

比较模型

RNA-seq：

作者首先通过RNA-seq比较了三种模型的的恶病质表征情况（control vs. cachetic patients)。分析它们的差异基因。 C-26, LLC， PDA分别找到了3474， 788， 141的差异基因。PDA模型中相关的基因都是与细胞胞外基质相关的。而前两种模型主要集中在RNA翻译水平的变化上。 有意思的是，PDA模型的差异基因在癌症病人中都是下调的，而在另外两种模型中85%都是上调的（C26),或者50%上调，50%下调。

RT-qPCR

作者还分析三种模型的肌肉自噬与蛋白酶体降解相关的基因表达，结果都呈现了不一致的现象。

  • autophagy gene: Atg5, Bnip3
  • E3 ligases: MuRF1, Atrogin-1

验证新模型：KPC GEMM of PDA

K-rasLSL.G12D/+; Trp53R172H/+; Pdx-1-Cre(KPC) 这也是一种非常常见的胰腺导管癌模型。它们发现KPC小鼠普遍比同龄小鼠个头小。 并且KPC的DEG只有一个，同样与PDA病人中得到的结果不一致。

构建新模型

基于前面转录分析的明显差异，作者因此想要构建一种新的模型。作者从以下几个方面进行考虑：

  • Model the progression of PDA
  • induce cancer in postnatal adults
  • establish a direct association between PDA and cachexia development: real tissue wasting are happening.
  • develop cachexia slower that the current model. 因此，作者在先用的KPC模型进行改进。基因型为：Kras+/G12D, Ptf1a+/ER-Cre, Ptenf/f。 作者随后进一步验证了这种模型，发现reporter gene在经过tamoxifen诱导后只在胰腺腺泡细胞中表达。 在end point期，癌症小鼠的组织学分析明显差异：
  • Acian Blue: 表征是否有粘蛋白或者粘多糖。 作者接下来想要评估是否KPP小鼠也患有恶液质。分别从体重、muscle masses(TA,tibialis anterior muscle，胫骨前肌；QUAD, quadriceps，四头肌；GAST, gastrocnemius腓肠肌)进行评价，发现都减少了。 。 此外，肌小节中actin和myosin的含量没有发生改变，KPP老鼠的muscle force降低，说明肌肉的功能受到损害。

此外，这种小鼠还呈现了一些其他多器官衰竭症状：脾脏肿大、心脏减小、白色脂肪和棕色脂肪降低，呼吸交换率（resting respiratory exchange ratio)增加。

作者还发现，在不同的时期进行诱导肿瘤，cachexia的发生也是渐进的。

最后，作者比较了KPP小鼠的转录组（control vs. tumor)，结果只发现了172个差异表达基因。并且Top5都是与胞外基质有关的基因，这一点与PDA病人的结果都是一致的。

结论

KPP小鼠可能比C26, LLC移植瘤模型更适宜用来模拟胰腺导管癌恶病质发生进程。

重新建立博客

2022年10月05日
博客
929 words 2 min read

前言

昨天我的脑子不知道出了什么问题，把自己两年前辛苦鼓捣出来的博客给毁了。准确来说是自己用了错误的方法去解决问题，导致这个问题像雪球一样越滚越大。于是我打算用一键删除，万事重做大法，天真的以为所有的的问题都会消失。结果问题不但没有消失，还越变越复杂。我不断地重新建立github repository，不断地尝试。总算有了一点眉目。希望能够在此详细的记录下自己重新建立个人网站的过程。以史为鉴吧。

流程

使用Rstudio安装blogdown

由于该R包是部署在github上的，因此我要解决的第一个问题就是能够翻墙访问github。通过一番查找，得知仍然是通过老套路把github的IP地址添加hosts文件下。Github的IP地址。。然后打开hosts文件。在下面添加

ssh IP Address @github.com

install.packages("blogdown");
library(blogdown)

使用blogdown新建website

选择New Project, New directory, 拖到最下面选择website with blogdown。在hugo-theme处填上自己喜欢的主题名字。 一般需要先在Hugo theme种去选择，然后找到对应的username/theme-name。这样一个初始的主题就建好了。

blogdown::build_site()#首先建立站点
blogdown::serve_site()#预览站点，用于本地调试用。

这一步会生成public文件夹，一定要注意没有生成。

在github新建仓库

在github上新建一个仓库，一定要注意仓库名字是username.github.io(不要随便取，这也是我刚开始忽略的点)。不要选择ReadMe.md和.gitignore文件。另一步需要安装git软件。略~~~~

把本地public文件夹上传到仓库

git init
git add remote origin http://caroxiaoyu.github.io.git # 将github远程库链接添加到本地
git commit -a
git push -u origin master

此时在github setting里面选择Pages，稍等一会儿能够看到如下界面： 。访问该站点，该博客就建立成功啦。

总结一些遇到的坑

  1. 需要修改hosts文件才能访问github。
  2. 整个过程不需要添加ssh密钥。
  3. git使用的3步：add, commit, push。如果仓库已经有东西了，这时不能再把本地的push过去，会产生冲突。
  4. 创建post时，draft要选择false，否则不会出现在public文件夹中。
